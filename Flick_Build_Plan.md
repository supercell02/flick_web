## DAY 3 — File Upload System

**Theme:** The core mechanic. Guests uploading photos is the whole product.

### Morning: UploadThing Backend

Paste PRD Section 4 into Claude Code. Implement:

**`src/app/api/uploadthing/core.ts`:**
- `eventCoverUploader` — single image, 4MB max, requires Clerk auth
- `guestMediaUploader` — images (10MB) + videos (50MB), anonymous, max 5 files, calls Convex `addUpload` on complete

## DAY 6 — Analytics + Security + Event Management

**Theme:** Instrumentation, safety rails, and the management view.

### Morning: Analytics (PostHog + Convex)

Paste PRD Section 7 into Claude Code:

1. `src/lib/posthog.ts` — initialize PostHog, export typed `track()` helper
2. `src/components/analytics/PostHogProvider.tsx` — wrap app in `layout.tsx`
3. Add tracking calls to all pages:
   - `event_created` after Convex createEvent
   - `qr_opened` on event page if `?src=qr` in URL
   - `gallery_opened` on every event page mount
   - `upload_started` when UploadZone begins
   - `upload_completed` in UploadThing callback
   - `download_gallery` on "Download All" click
   - `revisit` if localStorage flag is set

### Afternoon: Security + Event Management Page

**Security (PRD Section 10):**
- Upload rate limiting: 20 uploads/IP/hour/event (in-memory Map)
- File type allowlist: jpeg, png, webp, heic, mp4, mov, webm only
- Convex mutation guards: verify userId === event.hostId
- Slug collision retry (up to 5 attempts)
- Input sanitisation on title (max 100 chars), uploadedBy (max 50 chars)
- 500 upload cap per event

**Event Management (PRD Section 8):** Build `src/app/(protected)/dashboard/[eventId]/page.tsx`:
- Event summary section
- QRCodeCard (full size)
- Uploads table with "Hide" action (realtime via Convex)
- Analytics stat boxes (uploads, QR scans, gallery opens, unique visitors)
- Delete event with confirm dialog

### Day 6 Done When

## DAY 7 — Mobile Audit + Deployment

**Theme:** Polish every pixel at 375px. Ship to Vercel.

### Morning: Mobile Optimisation

Paste PRD Section 9 into Claude Code. Go through every page:

- All touch targets: minimum 44×44px (`min-h-11 min-w-11`)
- Upload zone: "Tap to add photos" on mobile, drag-and-drop on desktop
- Gallery: `columns-2` mobile → `md:columns-3` tablet → `xl:columns-4` desktop
- Typography: `text-3xl` → `text-6xl` for hero headline
- Navigation: MobileNav bottom bar on event pages
- `next/image` with proper `sizes` prop on all gallery images
- Skeleton loaders while Convex query loads
- OG meta tags on `/e/[slug]` pages

Test at 375px in Chrome DevTools. Fix every overflow and broken layout.

### Afternoon: Deployment

**Step 1: Vercel Configuration**

Create `vercel.json`:
```json
{
  "framework": "nextjs",
  "buildCommand": "npx convex deploy && next build",
  "outputDirectory": ".next"
}
```

Create `next.config.ts` additions:
```typescript
images: {
  remotePatterns: [
    { protocol: "https", hostname: "utfs.io" },
    { protocol: "https", hostname: "uploadthing.com" }
  ]
}
```

**Step 2: Deploy Convex to Production**

```bash
npx convex deploy --prod
```

Copy the production Convex URL — you'll need it for Vercel env vars.

**Step 3: Push to GitHub + Deploy on Vercel**

```bash
git init
git add .
git commit -m "feat: initial flick MVP"
git remote add origin https://github.com/YOUR_USERNAME/flick.git
git push -u origin main
```

Import repo on vercel.com → New Project.

**Step 4: Set Environment Variables on Vercel**

Add every variable from `.env.local`, but update:
- `NEXT_PUBLIC_APP_URL` → your Vercel domain (e.g. `https://flick.vercel.app`)
- `NEXT_PUBLIC_CONVEX_URL` → production Convex URL from `npx convex deploy --prod`

**Step 5: Build + Verify**

```bash
npm run build
```

Zero TypeScript errors. Zero ESLint warnings. Fix every one before deploying.

**Step 6: Write `README.md`**

See PRD Section 11 for contents. Include:
- 3-sentence overview
- Local setup steps
- Environment variable table
- Deploy instructions
- Links to Convex, Clerk, UploadThing dashboards

**Step 7: Create `env.example`**

Copy `.env.local`, replace all values with descriptions, commit to repo.

### Day 7 Done When

- Can create an event, share the link, upload from a phone
