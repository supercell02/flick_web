```

_id         (auto)

eventDate   number          Unix ms timestamp
revealAt    number          Unix ms timestamp (instant = eventDate, next day = eventDate + 86400000)
hostId      string          Clerk userId
createdAt   number          Date.now()
uploadCount number          default 0
isPublic    boolean         default true
revealMode  "instant" | "after_event" | "next_day"

── uploads ─────────────────────────────────────────────
_id         (auto)
eventId     Id<"events">
fileUrl     string
fileType    "image" | "video"
uploadedBy  string          optional guest name, default "Guest"
uploadedAt  number          Date.now()
size        number          bytes
deviceInfo  string          user-agent truncated to 200 chars
isVisible   boolean         default true (for soft moderation)

── analyticsEvents ─────────────────────────────────────
_id         (auto)
eventId     Id<"events"> | null
eventType   "event_created" | "qr_opened" | "upload_started" |
            "upload_completed" | "gallery_opened" | "revisit" |
            "download_gallery"
timestamp   number
metadata    any             JSON blob

Add indexes:
- events: by_slug (slug), by_host (hostId)
- uploads: by_event (eventId), by_event_time (eventId, uploadedAt)
- analyticsEvents: by_event (eventId), by_type (eventType)

After writing the schema, generate all Convex functions in:
- convex/events.ts  (createEvent, getEventBySlug, getEventsByHost, updateUploadCount)
- convex/uploads.ts (addUpload, getUploadsByEvent, deleteUpload)
- convex/analytics.ts (trackEvent, getAnalyticsByEvent)

Use proper Convex mutation/query/internalMutation patterns.
Validate all inputs. Never expose hostId to public queries.
```

## 10. Security & Abuse Prevention

### Prompt for Claude Code

```
Implement security measures throughout the app.

1. UPLOAD RATE LIMITING (UploadThing core.ts)
   In guestMediaUploader middleware:
   - Extract IP from request headers (x-forwarded-for or connection.remoteAddress)
   - Allow max 20 uploads per IP per hour per event
   - Use a simple in-memory Map (sufficient for MVP)
   - Return 429 error if limit exceeded

2. FILE TYPE VALIDATION (UploadThing core.ts)
   Allowlist only:
   Images: image/jpeg, image/png, image/webp, image/heic
   Videos: video/mp4, video/quicktime, video/webm
   Reject all others with clear error message.

3. SIGNED UPLOAD URLS
   UploadThing handles this by default. Verify this is enabled.

4. CONVEX MUTATION GUARDS
   In every Convex mutation that modifies event data:
   - Verify the calling userId (from Clerk) matches event.hostId
   - Throw ConvexError("Unauthorized") if not

5. SLUG COLLISION
   In createEvent mutation: if generated slug already exists, regenerate.
   Retry up to 5 times before throwing.

6. INPUT SANITISATION
   In createEvent: sanitize title (trim whitespace, max 100 chars)
   In addUpload: sanitize uploadedBy (trim, max 50 chars)
   In guestMediaUploader: truncate deviceInfo to 200 chars

7. MAX EVENT UPLOAD CAP
   In addUpload mutation: if event has ≥ 500 uploads, throw ConvexError("Upload limit reached")
   Show friendly error in UploadZone: "This gallery is full"
```

## 12. CLAUDE.MD (Persistent Context File)

Create this file at the root of the project for ongoing Claude Code sessions:

```markdown
# CLAUDE.md — Flick

## What This Project Is
Mobile-first web platform for collecting event photos/videos via QR code.
Guests upload without signing up. Hosts manage galleries behind Clerk auth.

## Stack
- Next.js 15 App Router + TypeScript
- Convex (DB + realtime)
- Clerk (host auth only)
- UploadThing (file storage)
- PostHog (analytics)
- shadcn/ui + TailwindCSS + Framer Motion

## Key Conventions
- Public routes: / and /e/[slug]
- Protected routes: /dashboard/* (Clerk required)
- Guests are always anonymous — never ask for auth on /e/* routes
- All DB access through Convex mutations/queries — no direct DB calls
- All uploads through UploadThing — never store files in Convex
- Track every key action in PostHog AND Convex analyticsEvents table
- Mobile-first: design for 375px, enhance for desktop

## File Structure
src/app/(public)/        ← public pages
src/app/(protected)/     ← dashboard (Clerk protected)
src/components/event/    ← event-specific components
src/components/layout/   ← navbar, footer, mobile nav
src/lib/                 ← utils, posthog, uploadthing helpers
convex/                  ← all DB schema + functions

## Naming Conventions
- Components: PascalCase
- Utilities: camelCase
- Convex queries: camelCase verbs (getEventBySlug, addUpload)
- Route files: page.tsx, layout.tsx (Next.js convention)

## Do Not Build (Post-MVP)
- Comments, likes, reactions
- Native mobile apps
- AI editing / highlight reels
- Social feed / discovery
- Livestreaming
- Facial recognition
```
